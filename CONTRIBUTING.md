# 贡献指南 / Contributing

欢迎进入结界。

单色结界的每一个项目，都是独立、自洽的系统。你的贡献——无论是代码、文档、设计，还是挑出毛病——都会让这些系统更完整。

## 我能做什么？

- **报告 Bug**：发现不符合预期的行为，请提交 Bug Report。
- **提出想法**：有希望在结界中看到的新东西，请提交 Feature Request。
- **提交代码**：直接动手实现，请遵循下方的 Pull Request 流程。

## 代码提交流程

1. **Fork** 目标仓库。
2. 在功能分支上工作：`git checkout -b feat/your-feature`
3. 保持提交信息清晰。一个提交做一件事，用中文或英文都可以，但要让人看得懂。
4. 如果改动涉及行为变化，请同步更新文档。
5. 发起 Pull Request，填写 PR 模板中的内容。

## 风格约定

- 不执着于某种具体的代码风格。**自洽优先**——和仓库已有的风格保持一致即可。
- 命名清晰，逻辑可读，不炫技。
- 测试不是强制的，但如果你改了关键逻辑，顺手补上测试会让人很放心。

## 行为准则

这里是一个安静的创作空间。带着尊重进来，带着成果离开。

- 讨论围绕事，不围绕人。
- 接受不同意见。结界欢迎审视和质疑。
- 不做破坏性、恶意或 spam 行为。

违反以上准则的发言或 PR 将被直接关闭。

所有贡献默认以项目根目录下 `LICENSE` 的条款授权。
